# Telegram Copywriter Bot — Agent System Design

**Date:** 2026-05-30
**Stack:** Python (aiogram) + Anthropic Claude API (prompt-chaining)
**Audience:** Single user (authenticated by TELEGRAM_USER_ID)

---

## Overview

Telegram-бот-копирайтер, который пишет в стиле владельца канала на основе выгруженных постов. Архитектура: динамическая маршрутизация через Оркестратора — он вызывает только нужных агентов в зависимости от типа запроса.

---

## Agent Team

### Оркестратор (Orchestrator)
- **Модель:** `claude-opus-4-8`
- **Температура:** `0.2`
- **Роль:** Точка входа для всех запросов из Telegram. Определяет интент пользователя, маршрутизирует к нужным агентам, пишет финальный пост/ответ в стиле пользователя, собирает результаты.
- **Инструменты:**
  - `route_to_agent(agent, task)` — передать задачу агенту
  - `read_style_profile()` — загрузить профиль стиля из `style_profile.json`
  - `write_post(brief)` — написать пост по заданию
  - `edit_post(original, instructions)` — отредактировать пост

### Архитектор (Architect)
- **Модель:** `claude-sonnet-4-6`
- **Температура:** `0.5`
- **Роль:** Стратег контента. Создаёт и управляет контент-планом, предлагает темы и серии постов на основе профиля стиля пользователя.
- **Инструменты:**
  - `read_content_plan()` — прочитать `content_plan.md`
  - `write_content_plan(plan)` — сохранить обновлённый план
  - `suggest_topics(style_profile, n)` — предложить N тем

### Декомпозитор (Decomposer)
- **Модель:** `claude-sonnet-4-6`
- **Температура:** `0.3`
- **Роль:** Аналитик стиля и задач. Парсит выгрузки Telegram-канала (MD/JSON), извлекает паттерны стиля (тон, длина, структура, любимые обороты), разбивает сложные задания на подзадачи.
- **Инструменты:**
  - `parse_telegram_export(file)` — разобрать MD или JSON выгрузку
  - `extract_style_patterns(posts)` → `style_profile.json`
  - `decompose_task(task)` → список подзадач

### Тестировщик (Tester)
- **Модель:** `claude-haiku-4-5-20251001`
- **Температура:** `0.1`
- **Роль:** Контролёр качества. Проверяет сгенерированный контент на соответствие стилю пользователя, краткость, человечность, хлёсткость. Возвращает оценку и конкретные замечания.
- **Инструменты:**
  - `check_style(post, style_profile)` → `{score: 0-10, issues: []}`
  - `approve_or_reject(score)` → `pass` (≥7) / `fail` + замечания
- **Лимит повторов:** при `fail` Оркестратор правит и повторяет максимум 2 раза; после — отдаёт лучший вариант с пометкой пользователю

## Data Flow

```
Пользователь (Telegram)
        │
        ▼
  [Оркестратор]  ← style_profile.json
        │
        ├─ "написать / отредактировать пост"
        │       └─► [Оркестратор пишет] ──► [Тестировщик]
        │                                         │
        │                                   pass ─► пользователь
        │                                   fail ─► [Оркестратор] правит → повтор
        │
        ├─ "контент-план"
        │       └─► [Архитектор] ──► [Декомпозитор] ──► сохранить content_plan.md
        │
        └─ "загрузить файл стиля" (.md / .json)
                └─► [Декомпозитор] ──► style_profile.json
```

## File Storage

| Файл | Назначение |
|---|---|
| `style_profile.json` | Извлечённые паттерны стиля (тон, длина, обороты) |
| `content_plan.md` | Текущий контент-план |
| `examples/` | Оригинальные выгрузки пользователя (MD/JSON) |

---

## Routing Logic (Orchestrator intent detection)

| Интент | Вызываемые агенты |
|---|---|
| Написать новый пост | Оркестратор → Тестировщик |
| Отредактировать пост | Оркестратор → Тестировщик |
| Показать / создать контент-план | Архитектор (→ Декомпозитор для сложных планов) |
| Загрузить примеры стиля | Декомпозитор |
| Сложная серия постов | Декомпозитор → Оркестратор → Тестировщик |

---

## Temperature Rationale

- **Оркестратор 0.2** — маршрутизация должна быть предсказуемой, письмо в стиле — контролируемым
- **Архитектор 0.5** — нужна лёгкая творческость для генерации тем и планов
- **Декомпозитор 0.3** — аналитическая задача, минимум галлюцинаций
- **Тестировщик 0.1** — строгая оценка, максимальная воспроизводимость

## Bot Commands (planned)

| Команда | Действие |
|---|---|
| `/start` | Приветствие, инструкция |
| `/plan` | Показать текущий контент-план |
| `/newplan` | Создать новый контент-план |
| `/write <тема>` | Написать пост на тему |
| `/edit` | Отредактировать переданный текст |
| `/upload` | Загрузить файл со стилем |
| `/style` | Показать текущий профиль стиля |
